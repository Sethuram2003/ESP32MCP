## Overview

This project implements an **MCP server on an ESP32** using Espressif's [`mcp-c-sdk`](https://components.espressif.com/) component. It connects to a Wi-Fi network, starts an HTTP transport, and exposes a `greet` tool that any MCP-compatible client (Claude Desktop, MCP Inspector, custom agent) can call over the network.

It is a minimal, end-to-end reference for embedding MCP into resource-constrained embedded devices.

## Features

- 📡 **Wi-Fi Station mode** with automatic reconnect
- 🔌 **MCP HTTP transport** — talk to the device with any MCP client
- 🛠️ **Tool registration** via the `esp-mcp-c-sdk` API (`esp_mcp_tool_create`, `esp_mcp_add_tool`)
- 🧱 **Modular component layout** — `wifi_station` and `mcp_server` are independent ESP-IDF components
- 🔁 **Event-driven** using FreeRTOS event groups for connection synchronization

## Prerequisites

- **ESP-IDF v5.x** — install from [docs.espressif.com](https://docs.espressif.com/projects/esp-idf/en/latest/esp32/get-started/)
- **A supported ESP32 board** (ESP32, ESP32-S2, ESP32-S3, ESP32-C3, etc.)
- **Python 3.8+** with `idf.py` on `PATH`
- A 2.4 GHz Wi-Fi network (SSID + password)

Set up the IDF environment in every new shell:

```bash
source $IDF_PATH/export.sh
```

## Getting started

### 1. Clone

```bash
git clone https://github.com/Sethuram2003/ESP32MCP.git
cd ESP32MCP
```

### 2. Configure Wi-Fi credentials

Edit `components/wifi_station/wifi_station.c` and replace the placeholder values:

```c
#define configWIFI_SSID      "myssid"
#define configWIFI_PASSWORD  "mypassword"
```

> For production, move these to `menuconfig` (Kconfig) and `sdkconfig.defaults` so credentials aren't baked into firmware images.

### 3. Build, flash, and monitor

```bash
idf.py set-target esp32          # or esp32s3, esp32c3, ...
idf.py build
idf.py -p /dev/ttyUSB0 flash monitor
```

You should see logs like:

```
WIFI_STATION: Got IP: 192.168.1.42
MAIN: Wi-Fi connected, proceeding to start MCP server...
MCP_SERVER_APP: MCP server initialized
MCP_SERVER_APP: MCP server started
```

### 4. Talk to the MCP server

The HTTP transport listens on the device's IP at the default MCP HTTP port. Use the [MCP Inspector](https://github.com/modelcontextprotocol/inspector) or any HTTP-capable MCP client, then invoke the registered tool:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": { "name": "greet", "arguments": {} }
}
```

Expected response:

```json
{
  "result": {
    "content": [{ "type": "text", "text": "Hello! This is your ESP32 MCP server." }]
  }
}
```

## Adding your own tools

Register new tools in `components/mcp_server/mcp_server.c`:

```c
static esp_mcp_value_t my_tool_cb(const esp_mcp_property_list_t *properties)
{
    (void)properties;
    return esp_mcp_value_create_string("custom result");
}

esp_mcp_tool_t *tool = esp_mcp_tool_create(
    "my_tool",
    "Describe what it does",
    my_tool_cb
);
ESP_ERROR_CHECK(esp_mcp_add_tool(s_mcp, tool));
```

Rebuild and flash — the new tool becomes available to any MCP client immediately.

## Configuration

| Option              | Where                                | Default                |
| ------------------- | ------------------------------------ | ---------------------- |
| Wi-Fi SSID          | `wifi_station.c` `configWIFI_SSID`   | `"myssid"` (placeholder) |
| Wi-Fi password      | `wifi_station.c` `configWIFI_PASSWORD` | `"mypassword"` (placeholder) |
| Wi-Fi connect timeout | `main/app_main.c`                    | `30000` ms             |
| Auth mode           | `wifi_station.c`                     | `WIFI_AUTH_WPA2_PSK`   |

## Troubleshooting

- **Stuck on `Wi-Fi init finished.`** — SSID/password mismatch, or 2.4 GHz network unavailable.
- **`Failed to connect to Wi-Fi within timeout`** — AP out of range or credentials wrong; double-check `wifi_station.c`.
- **Port already in use when flashing** — close any other serial monitor (Arduino IDE, `screen`, etc.).
- **`idf.py: command not found`** — re-run `source $IDF_PATH/export.sh`.
